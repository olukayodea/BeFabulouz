Authentication

To use the API, users must have the following:

      username: this is the username of the admin or API user on the server. this username is issued on registration and does not change
      secret: will be generated by befabulouz and issued to the user, this key will be subject to periodic changes and users will be advised via email of the new key before itâ€™s enforced
Thes authentication parameters will be supplied in the header and body of the API when it is used to uthenticate the user session

This concatenated strings of the key is to be encrypted with the SHA256 hashing algorithm before sending to the server

in your request headeer pass the fowing parameter

In sending a file to the server alongside every request, the following must be specified in clear text in the file. (see the method section for each method to see usage example)

curl --request GET | POST |PUT | DELETE \ --url https://befabulouz.com/api/<mode>/<action>/<value> \ --data '<JSON ] XML >' --header 'accept: application/json' --header 'content-type: application/json' --header 'username: username' --header 'secret: secret'

Users

Value Type Description
users_id int(11)
